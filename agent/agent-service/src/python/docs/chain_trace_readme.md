# 智能体链路追踪系统

## 概述

本系统提供了完整的智能体链路追踪功能，能够详细记录从协调智能体到规划智能体的整个执行过程，包括所有发送给LLM的prompt、LLM响应、数据流转等关键信息。

## 链路流程

### 1. 协调智能体阶段

#### 1.1 意图识别
- **FastText第一阶段**: 快速意图分类
- **Neo4j第二阶段**: 本体精确定位
- **LLM调用**: 发送完整prompt进行意图分析
- **日志输出**: 
  - 用户查询和可用工具信息
  - 完整的系统prompt
  - LLM响应结果
  - 意图识别结果和槽位信息

#### 1.2 上下文构建
- **RAG检索**: 从知识库检索相关内容
- **记忆检索**: 获取用户历史记忆
- **业务上下文**: 加载会话级和业务级上下文
- **日志输出**: 检索结果统计和上下文长度

#### 1.3 任务路由
- **智能体选择**: 根据意图路由到对应的规划智能体
- **Prompt获取**: 获取智能体特定的系统prompt
- **日志输出**: 路由决策和prompt信息

### 2. 规划智能体阶段

#### 2.1 目标拆解
- **LLM调用**: 发送目标拆解prompt
- **日志输出**: 
  - 智能体配置和能力范围
  - 完整的目标拆解prompt
  - LLM返回的子目标列表

#### 2.2 依赖分析
- **依赖图构建**: 分析任务间依赖关系
- **关键路径识别**: 找出关键执行路径
- **并行组识别**: 识别可并行执行的任务组
- **冲突检测**: 检测循环依赖等冲突
- **日志输出**: 分析结果统计和详细信息

#### 2.3 工具匹配
- **LLM调用**: 发送工具匹配prompt
- **日志输出**: 
  - 可用工具列表
  - 工具匹配prompt
  - LLM返回的匹配结果

#### 2.4 任务清单生成
- **任务创建**: 根据分析结果创建具体任务
- **执行策略**: 设置重试、超时等策略
- **日志输出**: 任务创建详情和统计信息

#### 2.5 执行引擎
- **工具调用**: 执行匹配的工具
- **重试机制**: 失败重试和回退工具使用
- **日志输出**: 
  - 执行过程详情
  - 重试和回退过程
  - 最终执行结果

### 日志特点
- ✅ 使用emoji图标增强可读性
- ✅ 每个阶段有清晰的开始和结束标识
- ✅ 所有LLM交互都有完整的prompt和响应
- ✅ 数据流转过程详细记录
- ✅ 错误信息突出显示
- ✅ 使用分隔线区分不同阶段

## 使用方法

### 1. 启用深度思考模式
在请求中设置 `thinking: true` 来启用完整的链路追踪：

```python
request = CoordinateRequest(
    user_id=1,
    query="查询停车场A今天的停车计费情况",
    session_id=None,
    thinking=True  # 启用深度思考模式
)
```

### 2. 查看日志
日志会输出到以下文件：
- `agent-service/logs/pg_agent_application.log`
- `logs/pg_agent_application.log`

### 3. 运行测试
```bash
cd agent-service/src/python
python test/chain_trace_test.py
```

## 关键日志输出点

### 协调智能体
1. **意图识别开始**: 用户查询、可用工具
2. **Prompt构建**: 系统prompt和用户查询
3. **LLM响应**: 意图识别结果
4. **上下文构建**: RAG和记忆检索结果
5. **任务路由**: 智能体选择和prompt获取

### 规划智能体
1. **目标拆解**: 智能体配置、拆解prompt、子目标结果
2. **依赖分析**: 依赖图、关键路径、并行组、冲突检测
3. **工具匹配**: 工具列表、匹配prompt、匹配结果
4. **任务生成**: 任务创建详情、执行策略
5. **执行引擎**: 工具调用、重试过程、最终结果

## 配置说明

### 日志级别
确保日志级别设置为INFO或DEBUG以查看完整的链路信息。

### 日志文件位置
在 `infrastructure/config/sys_config.py` 中配置日志文件路径。

## 故障排除

### 1. 日志不完整
- 检查日志级别设置
- 确认文件写入权限
- 检查磁盘空间

### 2. Prompt未显示
- 确认thinking模式已启用
- 检查LLM客户端配置
- 验证prompt服务状态

### 3. 阶段跳过
- 检查前置条件是否满足
- 验证依赖服务状态
- 查看错误日志

## 扩展功能

### 1. 添加新阶段
在相应的服务类中添加新的日志输出：

```python
self.log.info("=" * 80)
self.log.info("📍 [组件名称] 新阶段名称")
# 阶段逻辑和日志
self.log.info("=" * 80)
```

### 2. 自定义日志格式
可以修改emoji图标和分隔符来适应不同的显示需求。

### 3. 链路追踪持久化
可以将链路信息保存到数据库或文件中，便于后续分析。

## 性能考虑

- 详细的日志输出会影响性能，建议在生产环境中谨慎使用
- 可以通过配置开关来控制日志详细程度
- 考虑日志轮转和清理策略

## 总结

通过这个链路追踪系统，你可以：

1. **完整追踪**: 从用户查询到最终响应的完整过程
2. **调试分析**: 通过详细日志定位问题和优化性能
3. **Prompt调优**: 查看实际的prompt输入和LLM响应
4. **流程理解**: 深入理解智能体的工作流程和决策过程

这为智能体的开发、调试和优化提供了强大的工具支持。
